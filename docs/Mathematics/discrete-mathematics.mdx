# Discrete Mathematics

Combinatorics, graph theory, and discrete optimization for algorithms and HFT.

## Key topics

- **Combinatorics** — counting arrangements, the backbone of probability
- **Graph theory** — networks of nodes and edges
- **Discrete optimization** — best choice from a finite (often huge) set

## Why quants care

- **Combinatorics feeds probability:** binomial option-pricing trees literally count
  up-down path combinations; interview classics ("how many ways…") are combinatorics.
- **Graph theory** models market structure: currency-arbitrage detection is a
  negative-cycle search, settlement networks and contagion are graph problems.
- **Discrete optimization** appears wherever positions come in whole units: order
  routing, trade scheduling, and the algorithms behind **high-frequency trading** where
  data structures and O(·) complexity decide profitability.

## Try it — count binomial paths

<PySandbox title="Pascal's triangle prices options" code={`from math import comb

# In a binomial tree with n steps, how many paths end with k up-moves?
n = 10
print(f"paths through a {n}-step binomial tree:")
for k in range(n + 1):
    print(f"  {k:2d} up-moves: {comb(n, k):4d} paths  " + "#" * (comb(n, k) // 8))

# Total paths = 2^n
print("total:", sum(comb(n, k) for k in range(n + 1)), "= 2^10 =", 2**10)

# This bell shape is no accident: as n grows it becomes the normal
# distribution — and the binomial option price converges to Black-Scholes.`} />
